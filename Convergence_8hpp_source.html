23 /*!
24  * \ingroup LinearSolverGroup
25  * \brief Criteria that determine the linear solve has converged
26  *
27  * \details Most criteria are based on the residual magnitude
28  * \f$|r_k|=|b-Ax_k|\f$, where \f$x_k\f$ denotes the numerical solution after
29  * completion of iteration \f$k\f$ (see the \ref LinearSolverGroup
30  * documentation, `LinearSolver::Tags::Residual` and
31  * `LinearSolver::Tags::Magnitude`).
32  *
33  * The following criteria are implemented, ordered from highest to lowest
34  * priority:
35  *
36  * - AbsoluteResidual: Matches if the residual has reached this magnitude.
37  * - RelativeResidual: Matches if the residual has decreased by this factor,
38  * relative to the start of the first linear solver iteration.
39  * - MaxIterations: Matches if the number of iterations exceeds this limit.
40  *
41  * \note The smallest possible residual magnitude the linear solver can reach is
42  * the product between the machine epsilon and the condition number of the
43  * linear operator that is being inverted. Smaller residuals are numerical
44  * artifacts. Requiring an absolute or relative residual below this limit will
45  * likely lead to termination by `MaxIterations`.
46  *
47  * \note Remember that when the linear operator \f$A\f$ corresponds to a PDE
48  * discretization, decreasing the linear solver residual below the
49  * discretization error will not improve the numerical solution any further.
50  * I.e. the error \f$e_k=x_k-x_\mathrm{analytic}\f$ to an analytic solution
51  * will be dominated by the linear solver residual at first, but even if the
52  * discretization \f$Ax_k=b\f$ was exactly solved after some iteration \f$k\f$,
53  * the discretization residual
54  * \f$Ae_k=b-Ax_\mathrm{analytic}=r_\mathrm{discretization}\f$ would still
55  * remain. Therefore, ideally choose the absolute or relative residual criteria
56  * based on an estimate of the discretization residual.
57  */

116 /*!
117  * \brief Determine whether the \p convergence_criteria are met.
118  *
119  * \note This function assumes the \p iteration_id is that of the next, but
120  * not yet performed step. For instance, a `MaxIteration` criterion of 1 will
121  * match if the \p iteration_id is 1 or higher, since the first iteration
122  * (with id 0) has been completed. At this point, also the \p residual_magnitude
123  * reflects the state of the algorithm after completion of the first iteration.
124  * The `initial_residual_magnitude` always refers to the state before the first
125  * iteration has begun.
126  *
127  * \returns a `LinearSolver::ConvergenceReason` if the criteria are met, or
128  * `boost::none` otherwise.
129  */
130 boost::optional<ConvergenceReason> convergence_criteria_match(
131  const ConvergenceCriteria& convergence_criteria,
132  const IterationId& iteration_id, double residual_magnitude,
133  double initial_residual_magnitude) noexcept;

135 /*!
136  * \brief Signals convergence of the linear solver.
137  *
138  * \details Evaluates to `true` if the linear solver has converged and no
139  * further iterations should be performed. In this case, the `reason()` member
140  * function provides more information. If `false`, calling `reason()` is an
141  * error.
142  *
143  * The stream operator provides a human-readable description of the convergence
144  * status.
145  *
146  * This type default-constructs to a state that signals the linear solver has
147  * not yet converged.
148  */

156  HasConverged(const ConvergenceCriteria& convergence_criteria,
157  const IterationId& iteration_id, double residual_magnitude,
158  double initial_residual_magnitude) noexcept;
159 
160  explicit operator bool() const noexcept { return static_cast<bool>(reason_); }

162  /*!
